# 项目约束

## 设计原则
- ✅ 极简主义：保持文件数量最小化
- ✅ Git 友好：纯文本，可 diff，可跨设备同步
- ✅ 人类可读：Markdown 优先，避免复杂格式

## 技术约束
- 必须：Bash 兼容（macOS/Linux）
- 必须：无外部依赖（纯 Claude Code 功能）
- 禁止：复杂的 JSON 解析（保持简单）

## 用户约束
- 目标用户：个人开发者
- 使用场景：单人项目，多任务切换
- 心智负担：核心命令不超过 3 个
- 信息过载：智能建议数量 2-5 条（避免决策疲劳）

## 定位约束
- 核心定位：AI 对话 → 项目资产的转换器
- 不做万能工具：不复制 Linear（项目管理）、Notion（知识管理）、Obsidian（知识图谱）的功能
- 独特价值：与 Claude Code 深度集成 + 自动从对话中提取 + Git 友好

## 历史包袱
- 无（新项目）
