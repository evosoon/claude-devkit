# /save — 锚定当前上下文

回顾本次对话，将隐性决策和工作状态沉淀到项目文件中。

## 执行步骤

1. **创建目录**（如不存在）：`mkdir -p .claude/context/`

2. **回顾对话**，提取以下信息：
   - 当前任务是什么
   - 做了哪些工作，完成了什么，还剩什么
   - 做了哪些决策（选了什么方案、放弃了什么、为什么）
   - 有什么阻塞或待解决问题
   - 下一步具体动作

3. **覆写** `.claude/context/state.md`，严格使用以下格式（≤30 行）：

```markdown
## 当前任务
<一句话描述当前在做什么>

## 进度
- [x] 已完成项
- [ ] 未完成项

## 阻塞
<阻塞项，无则写"无">

## 下一步
<下次开始时应该做的第一件事>
```

4. **追加**新决策到 `.claude/context/decisions.md`。只记录本次对话中**新产生的**决策，不重复已有记录。每条决策使用以下格式追加到文件末尾：

```markdown
## YYYY-MM-DD <决策标题>
- **选择**: <选了什么>
- **放弃**: <弃了什么>
- **原因**: <为什么这样选>
```

如果本次对话没有产生新决策，不追加任何内容。

5. **输出确认**，用中文简要说明保存了什么（2-3 行即可）。

## 注意事项

- state.md 每次**覆写**，永远反映最新状态
- decisions.md 只**追加**，不修改已有内容
- 提取决策时关注"为什么"而非"做了什么"——做了什么在 git 里有，为什么不记就丢了
- 如果是项目首次使用 /save，在确认中提示用户将 `.claude/context/` 加入 git 跟踪
